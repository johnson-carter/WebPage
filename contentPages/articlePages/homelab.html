- Home labbing is a fascinating hobby that allows enthusiasts to build and manage their own servers, networks, and applications at home. It provides a hands-on way to learn about IT infrastructure, software development, and system administration. Starting today, we will cover the basics to get your server up and running!


Key Aspects of Home Labbing:

  1. Setting up your server
  2. Network configuration & remote access
  3. Configuring applications

1. Setting up your server

- You may think setting up your own server will be difficult; there is some truth to that, but this guide will make home-labbing a breeze.


- First, we need to determine what hardware our server will run on. My personal favorite choice for this step is finding an old laptop/PC. Using recycled tech is great because it allows you to enter the field of home-labbing without making a steep initial investment. This guide assumes you have some (nothing too advanced) knowledge of computer science. For your first machine or a hobby project, high-end specs are not important. A powerful processor, high amounts of memory, and lots of storage are key performance variables.


- Secondly, we need to choose an operating system. For this guide, I will be using Ubuntu Server LTS 24 due to it's wide library support and reputation for security. This guide will work with any server OS (in theory), and be especially similar for similar Debian based systems. Another important thing to note; installing a version of the operating system without a GUI will save precious resources on the host machine, so getting used to using the terminal is can be very beneficial here. Flash the OS to a storage device and configure your host device to boot into the storage medium. Installation is straight-forward, but here are a few pointers:

  • LVM Group - This is useful for partitioning your drive, and is enabled by default. For the purposes of this guide, it will make our job harder; disable it.
  • SSH - Be sure to install dependencies for SSH when prompted. This will allow us to access the server from an external device.
  • Device name, user name, and password - Remember to write this down! This is how we will access the PC later

Now your server should be up and running. Let's add some features.

2. Network configuration and remote access

- Now that the server is running, we need to configure some things that will make interfacing with the device easier. The first step in this process is to assign a static IP to your host machine. This means that when we talk to the device from the same network, it always has the same location. This will require you to log into your router's access dashboard (Usually found at 192.168.1.1). On my router, the setting was under a tab called DHCP reservation, but this will differ from model to model.


- Next we need to enable remote access. Using SSH or Secure Shell protocol, we can access the server host machine via a command prompt on our main device. Open command prompt, powershell, or your favorite terminal and type in the following:

$ ssh [Username you configured]@192.168.1.[Local IP you configured]

- Enter the command and login as prompted. Once complete, you have successfully gained access to your host machine and can begin adding functionality!

3. Configuring Applications

- With your server up and running, access protocol established, and network settings configured, it is now time to create the server of your dreams. This is where you own vision for a home-server takes over. To get you started here are some common applications:

Server Management Tools

- These tools are commonly used to make the management and deployment of server resources more simple

  • Docker: Create instances of programs that run in a container, here the exact environment variables can be meticulously managed and transferred between devices. If you prefer a GUI based experience, look at a web-hosted dashboard like Portainer
  • Git: The quintessential version control software. Learning how Git and GitHub works will prove valuable in your network admin journey.

Self-Hosted Alternatives

  • NextCloud & File Browser: Open source alternative to Google Drive/Apple ICloud/Microsoft OneDrive/Dropbox
  • JellyFin & Plex: Open source alternatives to media hosting platforms like Netflix and Hulu
  • Pi-Hole: DNS sinkhole and VPN software
  • Ollama: Run customizable, open source, and portable LLM models. Offline and private.
